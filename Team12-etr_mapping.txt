// 팀 별 ER Schema가 어떻게 관계형 Schema로 매핑 되었는지 관계 순으로 설명할 것.
// ER Schema에 수정사항이 있으면 반드시 명시할 것

// TRANSLATED의 id 소문자 -> 대문자로 바꿔야함 

// 1. weak entity에 partial key를 표기해주어야 합니다.(-1)
// 2. Entity에 대한 설명만 기술되어 있고 attribute와 relationship 설명 부족(-1)
// 3. weak entity와 관계를 맺을 때는 identifying relationship, total participation으로 표현해야합니다(-1).

0. ER Schema 수정사항
1) Rating entity에 partial key인 rating_no 추가. 

1. entity type을 관계형 Schema로 매핑

1) ACCOUNT 

- Account entity를 ACCOUNT Relation에 매핑하였음.  
- Account entity의 key attribute인 ID를 ACCOUNT Relation의 기본키로 매핑하였음. 

2) PUBLISHER 

- Publisher entity를 PUBLISHER Relation에 매핑하였음.  
- Publisher entity의 key attribute인 ID를 PUBLISHER Relation의 기본키로 매핑하였음. 

3) TRANSLATOR
- Translator entity를 TRANSLATOR Relation에 매핑하였음.  
- Translator entity의 key attribute인 ID를 TRANSLATOR Relation의 기본키로 매핑하였음. 

4) AUTHOR
- Author entity를 AUTHOR Relation에 매핑하였음.  
- Author entity의 key attribute인 ID를 AUTHOR Relation의 기본키로 매핑하였음. 

5) GENRE
- Genre entity를 GENRE Relation에 매핑하였음.  
- Genre entity의 key attribute인 ID를 GENRE Relation의 기본키로 매핑하였음. 

6) BOOK
- Book entity를 BOOK Relation에 매핑하였음.  
- Book entity의 key attribute인 ISBN을 BOOK Relation의 기본키로 매핑하였음. 

7) ADMIN
- Admin entity를 ADMIN Relation에 매핑하였음.  
- Admin entity의 key attribute인 ID를 ADMIN Relation의 기본키로 매핑하였음. 

2. weak entity type을 관계형 Schema로 매핑

1) RATING 
- Rating entity를 RATING Relation에 매핑하였음. 
- BOOK의 기본키인 ISBN을 RATING의 외래키 Book_ID로 설정하였음.(Constraints : ON DELETE CASCADE)
- ADMIN의 기본키인 ID를 RATING의 외래키 Account_ID로 설정하였음.(Constraints : ON DELETE CASCADE)
- 외래키 Book_ID와 외래키 Account_ID의 복합키를 기본키로 설정하였음. 


3. 1:N Relation을 관계형 Schema에 매핑

1) Evaluate Relationship
- ADMIN(1)의 기본키인 ID를 RATING(N)의 외래키 Account_ID로 설정하였음. 

2) Contain Relationship 
- BOOK(1)의 기본키인 ISBN을 RATING(N)의 외래키 Book_ID로 설정하였음.


4. M:N Relation을 관계형 Schema에 매핑

1) MANAGES
- Manages Relationship을 MANAGES Relation에 매핑하였음. 
- ACCOUNT의 기본키인 ID를 MANAGES의 외래키 Admin_ID로 설정하였음. 
- BOOK의 기본키인 ISBN을 MANAGES의 외래키 Book_ID로 설정하였음.
- 외래키인 Admin_ID와 Book_ID의 복합키를 기본키로 설정하였음. 

2) PUBLISHED_BY
- Published_by Relationship을 PUBLISHED_BY Relation에 매핑하였음. 
- PUBLISHER의 기본키인 ID를 PUBLISHED_BY의 외래키 Pub_ID로 설정하였음. 
- BOOK의 기본키인 ISBN을 PUBLISHED_BY의 외래키 Book_ID로 설정하였음.
- 외래키인 Pub_ID와 Book_ID의 복합키를 기본키로 설정하였음. 

3) TRANSLATED 
- Translated Relationship을 TRANSLATED Relation에 매핑하였음. 
- TRANSLATOR의 기본키인 ID를 TRANSLATED의 외래키 Translator_ID로 설정하였음. 
- BOOK의 기본키인 ISBN을 TRANSLATED의 외래키 Book_ID로 설정하였음.
- 외래키인 Translator_ID와 Book_ID의 복합키를 기본키로 설정하였음. 

4) AUTHORED 
- Authored Relationship을 AUTHORED Relation에 매핑하였음. 
- AUTHOR의 기본키인 ID를 AUTHORED의 외래키 Author_ID로 설정하였음.(Constraints : ON DELETE CASCADE)
- BOOK의 기본키인 ISBN을 AUTHORED의 외래키 Book_ID로 설정하였음.
- 외래키인 Author_ID와 Book_ID의 복합키를 기본키로 설정하였음. 

5) BELONG 
- Belong Relationship을 BELONG Relation에 매핑하였음. 
- GENRE의 기본키인 ID를 BELONG의 외래키 Genre_ID로 설정하였음. 
- BOOK의 기본키인 ISBN을 BELONG의 외래키 Book_ID로 설정하였음.
- 외래키인 Genre_ID와 Book_ID의 복합키를 기본키로 설정하였음. 

6) BORROW 
- Borrow Relationship을 BORROW Relation에 매핑하였음. 
- ADMIN의 기본키인 ID를 BORROW의 외래키 Account_ID로 설정하였음. 
- BOOK의 기본키인 ISBN을 BORROW의 외래키 Book_ID로 설정하였음.
- 외래키인 Account_ID와 Book_ID의 복합키를 기본키로 설정하였음.
